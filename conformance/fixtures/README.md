# conformance/fixtures/

Captured wire transcripts from real Claude Code sessions against the
in-tree cMCP servers. Two purposes:

1. **Regression bank** — every transcript here was, at one point, a
   shape of traffic that exercised a real interaction. The replay-based
   gate that closes phase 5.3 will replay each fixture's `dir:"in"`
   frames at the server and assert the `dir:"out"` frames match (modulo
   timing). Anything that breaks an existing transcript without a
   deliberate spec / API change is a regression.

2. **Failure archive** — when a playbook task surfaces an interesting
   bug, the transcript that triggered it is dropped here with a name
   that explains what it caught. Even after the fix, the fixture stays
   — guarantees the bug doesn't quietly come back.

## Format

One JSONL file per session. Generated by `tools/cmcp-tee/cmcp-tee` —
see [`../claude_code_setup.md`](../claude_code_setup.md). Each line:

```jsonl
{"t":<unix-seconds>,"dir":"in"|"out","frame":"<raw JSON-RPC frame>"}
```

## Layout

```
fixtures/
├── echo-server/
│   └── <symptom>.jsonl
├── filesystem-mcp/
│   └── <symptom>.jsonl
└── crag-mcp/
    └── <symptom>.jsonl
```

Naming: `<symptom>.jsonl`. Examples:

- `fs_read_offset_off_by_one.jsonl`
- `add_negative_overflow.jsonl`
- `crag_unicode_query_mangled.jsonl`

Group by **symptom**, not by Claude Code session date — when the gate
fires, the reader wants the bug class, not the calendar.

## Adding a fixture

1. Run the session through cmcp-tee (see setup doc).
2. Trim the JSONL down to just the conversation that demonstrates the
   bug — start at `initialize`, end at the last frame of the failing
   request. Drop unrelated turns.
3. Save under the relevant server's directory with a symptom name.
4. Commit. If the bug is unfixed, also file the issue against the
   right layer per the triage table in the setup doc.
